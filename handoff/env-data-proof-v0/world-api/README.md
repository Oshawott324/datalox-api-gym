# Datalox World API v0

This is a handoff-level API preview. It is intentionally kept outside `src/`
until the collaborator validates the interface.

The API separates four concerns:

```text
WorldDriver       reset / tools / step / finalize / export
Runtime backend   runnable-world, sandbox command, replay fixture, etc.
Transport adapter MCP, Python, CLI, later HTTP
Evidence layer    trajectory and replay artifacts emitted by the backend
```

MCP is a compatibility adapter. The canonical world interface is the
reset/step/finalize/export lifecycle.

## Python Smoke

```bash
PYTHONPATH=handoff/env-data-proof-v0/world-api \
python3 -m datalox_world.adapters.cli smoke \
  --world handoff/env-data-proof-v0/world-api/world.spec.json \
  --task-id fastq-qc-nanopore-fail-001 \
  --run /tmp/datalox-fastq-world-api
```

## MCP Stdio Preview

```bash
PYTHONPATH=handoff/env-data-proof-v0/world-api \
python3 handoff/env-data-proof-v0/world-api/datalox_world/adapters/mcp_stdio.py \
  --world handoff/env-data-proof-v0/world-api/world.spec.json \
  --task-id fastq-qc-nanopore-fail-001 \
  --run /tmp/datalox-fastq-mcp
```

The MCP adapter starts one task session and exposes the environment tool
catalog plus `datalox_submit_answer`. The task prompt scopes which tools are
relevant for the current task.

SFT message exports use OpenAI Chat Completions-compatible rows:

```text
messages + tools + tool_choice + provider_format
```

The system prompt carries task policy, evidence requirements, final-answer
contract, and task-relevant tool names/descriptions. Full JSON input schemas
stay in the row-level `tools` field instead of being baked into the prompt.

## Boundary

This package is not a sandbox platform and not a new domain runtime. A sandbox
or domain environment plugs in through a driver. Datalox API Gym should
record and export the evidence generated by those worlds.
