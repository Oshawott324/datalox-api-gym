Zheng Feedback Synthesis

A consolidated read of Zheng's feedback on Datalox positioning, roadmap, and what would make the environment work valuable to training teams.

Drafted for internal product alignment. If this is pushed to a public repository, consider renaming it to external-research-feedback-synthesis.html or removing personal attribution.

One-Line Read

Zheng is pushing Datalox away from an RL-only framing and toward a practical environment/data product: build useful task environments that can generate many grounded trajectories, export SFT/eval data cleanly, and leave reward design plus final training formats adjustable by training teams.

Product correction: the moat is not "we record trajectories." If tools and returns are shallow, LLMs can synthesize trajectories quickly. The moat has to be real environment quality: state, tools, validators, sample data, failure modes, scaffold diversity, and enough sub-environments to train broader capability.

AppWorld reference

 Agent scaffolds change the trajectory format. A single trajectory schema cannot serve all training styles equally. Represent scaffold specs explicitly. Let the same task environment be run under different scaffolds, then export scaffold-specific frames.
这些其实应该主要是训练方来考虑的事情。
 Training teams should decide exact trajectory shape, loss, reward, and model-visible formatting. Datalox should provide raw grounded evidence plus clean adapters, not force a single opinionated trainer format.
如果只是造轨迹，定义好一些 tool 还有返回值，用大模型造轨迹也是比较快的。
 Simple tool schemas and synthetic returns are commoditizable. They are not enough for a durable product wedge. The environment must include hard-to-fake value: domain sample data, deterministic algorithms, validators, structured state transitions, failure cases, and replayable evidence.
你们的环境有没有什么不可替代性？
 This is the central product challenge. Why use Datalox instead of asking an LLM to generate fake trajectories? Prove non-substitutability through real domain environments such as FlowCyto, molecule biology, and protein visualization where tool outputs depend on actual files, state, and validators.
如果目标是提升 agent 的通用能力，将一个环境做得非常深，非常完善，可能不如制造大量弱一点的环境。
 General capability may benefit more from breadth across many task worlds than from one perfect vertical environment. Roadmap should include many small, deterministic sub-environments and fixture sets, not only one deep FlowCyto proof.
环境数量提升到一定程度后，通用 agent 能力可能会有一个涌现。
 Environment diversity may be a scaling law for agent behavior. Track environment count, task family diversity, scaffold diversity, failure coverage, and export volume as product metrics.
可以把你们的环境改造一下，比如生成多个子环境，提升生物应用领域通用 agent 能力。
 Bio-domain generality can come from many related sub-environments rather than one monolithic tool. Split domain MCPs into environment packs: flow cytometry gating, QC recovery, compensation checks, sequence import, plasmid annotation, protein visualization, mutation inspection, etc.
很多客户应该没有那么多算力资源，低资源还是得 SFT。
 Many users will not have enough compute for heavy RL. SFT is the realistic first training path. Open-source copy should say "SFT-first" or "compute-constrained teams" rather than implying every customer will run RL.

Updated Product Frame

The strongest frame after this feedback is:

Datalox provides versioned agent task environments and the replay/data layer that turns environment runs into SFT, eval, preference, and RL-ready artifacts.

This avoids three traps:

  • It does not claim Datalox owns the trainer.
  • It does not claim Datalox owns every reward function.
  • It does not reduce the product to a trajectory recorder, which is too easy to substitute.

Roadmap Changes

Change Concrete Work Pass Criteria
SFT-first path Make datalox run and datalox export sft the first visible path for fixture sets. One command produces run.json, transcript.jsonl, and one or more valid sft_frame.v1 rows.
Reference reward only Each fixture set may include verifier metadata and a reference reward recipe, but no fixed universal reward engine. A training team can ignore or replace reward settings without breaking fixture install/run/export.
Scaffold-aware exports Add scaffold specs for ReAct, plan-and-execute, and direct tool-use variants where useful. The same task can produce different model-visible completion frames based on scaffold choice.
Many small environments Create multiple small bio sub-environments instead of only one deep FlowCyto vertical. Catalog shows multiple task families, each with sample data, tool contracts, failure cases, and at least one export path.
Non-substitutability proof For each environment pack, document what cannot be cheaply faked by synthetic tool logs. Each pack has at least one validator or state transition tied to real domain artifacts or deterministic domain computation.

Repo Ownership Implications

Repo Owns Should Not Own
datalox-agent-replay Fixture install/run/replay/export engine, model endpoint adapter, replay miss behavior, SFT/eval/preference adapters. Scientific algorithms, live domain UI, universal reward design, RL training orchestration.
datalox-replay-fixtures Published fixture catalogs, fixture sets, task specs, scaffold specs, verifier metadata, archived replay bundles. Live MCP runtime implementation or trainer-specific data transformations beyond declared adapters.
Domain MCP repos Live domain environments, sample data, deterministic domain tools, validators, workspace state, agent-native tool contracts. Central replay engine or trainer-specific export policy.

Immediate Next Steps

  1. Finish the visible FlowCyto proof as an agent-native reconstruction of a traditional workflow: structured workspace, typed tools, deterministic validation, replay evidence, and SFT export.
  2. Turn FlowCyto into multiple small sub-environments, not one oversized demo.
  3. Add scaffold metadata so the same environment can generate ReAct-style and plan-and-execute-style trajectories.
  4. Write one "non-substitutability" section for each environment pack.
  5. Update public copy to lead with agent-native workflow environments and SFT/eval data generation, with RL as a downstream consumer path.

Copy Guidance

Prefer this wording:

Datalox turns traditional workflows into agent-native environments, then packages replay evidence into grounded SFT and eval data.

Avoid this wording:

Datalox is an Agentic RL reward engine.
Datalox defines the universal trajectory format.
Datalox records trajectories, therefore it is the product.
Datalox guarantees identical full agent trajectories in complex live environments.
